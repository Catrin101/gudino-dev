# I18N.md — Sistema de Internacionalización (ES / EN)

## Objetivo

Implementar un sistema de cambio de idioma **sin recarga de página** y sin depender del traductor del navegador. El usuario elige ES o EN desde la navbar y toda la página cambia instantáneamente.

---

## Arquitectura del Sistema

### Cómo funciona

1. Todo el contenido textual vive en un objeto JS `translations` con claves `es` y `en`
2. Los elementos HTML que tienen contenido traducible llevan el atributo `data-i18n="clave.ruta"`
3. Al cambiar idioma, una función recorre todos los elementos con `data-i18n` y sustituye su contenido
4. El idioma activo se guarda en `localStorage` para persistir entre visitas
5. Al cargar la página, se lee `localStorage` y se aplica el idioma guardado (default: `es`)

---

## Implementación

### 1. Atributo en HTML

```html
<!-- Texto simple -->
<h1 data-i18n="hero.headline">Construimos sistemas web...</h1>
<p  data-i18n="hero.subheadline">Desarrollo de software a medida...</p>

<!-- Placeholder de input -->
<input type="text" data-i18n-placeholder="contact.form.name" placeholder="Tu nombre">

<!-- Atributo title -->
<a href="..." data-i18n-title="nav.services" title="Servicios">...</a>
```

### 3. Motor de i18n (`assets/js/utils/i18n.js`)

```js
// Estado global del idioma
let currentLang = localStorage.getItem('gudino-lang') || 'es';

/**
 * Aplica el idioma seleccionado a todos los elementos con data-i18n
 */
function applyLanguage(lang) {
  currentLang = lang;
  localStorage.setItem('gudino-lang', lang);

  // Actualizar botones del selector
  document.querySelectorAll('.lang-btn').forEach(btn => {
    btn.classList.toggle('active', btn.dataset.lang === lang);
  });

  // Actualizar atributo lang del HTML
  document.documentElement.lang = lang;

  // Re-renderizar componentes dinámicos con el nuevo idioma
  renderServices(lang);
  renderProjects(lang);
  renderAbout(lang);
  renderStack(lang);
  renderContact(lang);
  renderNav(lang);
  renderHero(lang);
  renderFooter(lang);
}

/**
 * Helper: obtiene valor anidado de objeto por ruta de string
 * Ejemplo: getNestedValue(translations.es, 'hero.headline')
 */
function t(key, lang = currentLang) {
  return key.split('.').reduce((obj, k) => obj?.[k], translations[lang]) ?? key;
}

/**
 * Inicialización
 */
function initI18n() {
  // Eventos del selector de idioma
  document.querySelectorAll('.lang-btn').forEach(btn => {
    btn.addEventListener('click', () => applyLanguage(btn.dataset.lang));
  });

  // Aplicar idioma guardado al cargar
  applyLanguage(currentLang);
}
```

## HTML del Selector de Idioma (en el navbar)

```html
<div class="lang-switcher" role="group" aria-label="Seleccionar idioma / Select language">
  <button class="lang-btn" data-lang="es" aria-label="Español">ES</button>
  <button class="lang-btn" data-lang="en" aria-label="English">EN</button>
</div>
```

---

## Notas de Implementación

- Los componentes dinámicos (servicios, proyectos, etc.) se **re-renderizan completamente** al cambiar idioma, no se usan atributos `data-i18n` en ellos porque su contenido es generado por JS.
- Los elementos estáticos del HTML (navbar links, hero, footer) sí pueden usar `data-i18n` si se prefiere un enfoque mixto.
- El cambio de idioma debe ser **instantáneo** — no animar ni hacer fade, solo reemplazar texto.
- El selector de idioma debe ser visible en **mobile** también (puede simplificarse a iconos de bandera o solo "ES | EN").

---

## Accesibilidad

- El `<html>` debe tener `lang="es"` o `lang="en"` actualizado dinámicamente
- Los botones del selector tienen `aria-label` descriptivo
- El cambio de idioma no mueve el foco del usuario ni recarga la página
